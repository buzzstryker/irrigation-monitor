# Migration: SQLite/Lenovo → Supabase + Railway

## Overview

The irrigation-monitor project is migrating from local SQLite (Lenovo laptop) to cloud infrastructure. This migration enables:

- **Continuous polling** regardless of laptop sleep/travel
- **Dashboard accessible from anywhere** (not just localhost)
- **Reliable storage** for the upcoming Tuya tank sensor integration
- **Separation of concerns** between data storage and compute

### Stack

| Component | Current (local) | Target (cloud) |
|-----------|----------------|----------------|
| Database | SQLite (irrigation.db) | **Supabase** (Postgres) |
| Poller | Lenovo pm2 (irrigation-poll) | **Railway** (always-on Node.js) |
| API Server | Lenovo pm2 (irrigation-server) | Railway or Vercel (TBD Phase 9) |
| Dashboard | http://localhost:3001 | TBD (Vercel likely, Phase 9) |
| Tank Sensor | Not yet integrated | Tuya Cloud API → Supabase (Phase 8) |

### Key Schema Translations (SQLite → Postgres)

| SQLite | Postgres | Notes |
|--------|----------|-------|
| `INTEGER AUTOINCREMENT` | `BIGINT GENERATED BY DEFAULT AS IDENTITY` | Modern identity syntax |
| `INTEGER` (timestamp) | `BIGINT` | Unix epoch seconds, future-proof |
| `REAL` | `DOUBLE PRECISION` | Floating-point |
| `TEXT` | `TEXT` | Unbounded string |
| `DATETIME` | `BIGINT` | Normalized to Unix epoch for consistency |
| `DEFAULT (unixepoch())` | `DEFAULT (EXTRACT(EPOCH FROM NOW())::BIGINT)` | Current timestamp |
| `CHECK (gpm >= 0)` | `CHECK (gpm >= 0)` | Same constraint syntax |

## Phase 2: Supabase Setup (MANUAL — your turn)

**Estimated time:** 10 minutes  
**Prerequisites:** Supabase account (free tier is sufficient)

### Steps

1. **Create Supabase project**
   - Go to https://supabase.com
   - Sign up or log in
   - Click **"New Project"**
   - Project name: `irrigation-monitor`
   - Database password: Generate a strong one, **save it in your password manager**
   - Region: **US West 2** (closest to Loomis, CA)
   - Pricing plan: **Free tier** (sufficient for this workload)
   - Click **"Create new project"**
   - Wait ~2 minutes for provisioning

2. **Copy API credentials**
   - Once project is ready, go to: **Project Settings → API**
   - Copy and save the following (you'll need them in Phase 4):
     - **Project URL:** `https://[PROJECT-REF].supabase.co`
     - **Project anon key:** (long string starting with `eyJ...`)
     - **Service role key:** (separate, longer string — **keep secret!**)
   - Store these in your password manager or a secure note

3. **Apply schema**
   - In Supabase dashboard, go to: **SQL Editor → New Query**
   - Open `migrations/supabase_schema.sql` from this repo in a text editor
   - Copy the **entire contents** (it's a single transaction)
   - Paste into the Supabase SQL Editor
   - Click **"Run"**
   - Should complete in a few seconds with no errors
   - Expected result: "Success. No rows returned"

4. **Verify tables created**
   - Go to: **Table Editor**
   - Verify all **16 tables** are listed in the left sidebar:
     - ditch_health_log
     - et_log
     - flow_calibration_log
     - observations
     - scheduled_reminders
     - sms_log
     - tank_level_log
     - tank_sensor_log
     - user_preferences
     - warnings
     - watering_events
     - zone_coefficients
     - zone_daily_analysis
     - zone_gpm_change_log
     - zone_gpm_overrides
     - zone_state_log
   - Click on a few tables to verify columns are present

### Troubleshooting

- **Error: relation already exists** → You've already run the schema. Drop the tables and re-run, or skip to Phase 3.
- **Syntax error** → Copy-paste issue; ensure you copied the entire file including BEGIN and COMMIT.
- **Timeout** → Supabase free tier can be slow; wait 30 seconds and try again.

### Import Process

For each CSV file in `migrations/data_export/`, upload via Supabase Studio:

1. **Open Table Editor** in Supabase dashboard
2. **Click on the table name** in the left sidebar (e.g., `zone_state_log`)
3. Click **"..." menu** (three dots) in top right → **"Insert" → "Import data from CSV"**
4. **Upload the CSV file** (e.g., `zone_state_log.csv`)
5. **Verify column mappings** — Supabase should auto-detect columns from CSV header
6. Click **"Import"**
7. Wait for upload to complete (large tables may take 10-30 seconds)
8. **Verify row count** matches `MANIFEST.txt`

## Rollback Plan

If migration fails at any phase:

1. **Before Phase 6 cutover:** Simply revert code to commit before Phase 4 refactor, continue using local SQLite
2. **After Phase 6 cutover:** Stop Railway service, restart Lenovo pm2 services, re-enable SQLite mode
3. **Data loss prevention:** Keep local SQLite database as backup until Phase 7 burn-in is complete

**Backup strategy:**
- Take a snapshot of `irrigation.db` before Phase 6
- Store in `backups/irrigation_pre_migration_YYYY-MM-DD.db`
- Keep for 30 days post-migration
