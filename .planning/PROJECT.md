# Project: irrigation-monitor

## What We're Building

An ET-driven residential irrigation control system that replaces static Hydrawise programmed schedules with dynamic, software-owned scheduling and an SMS observation feedback loop for landscaper/owner observations.

**System Overview:**
- Monitors 3 Hydrawise controllers (Garage, Pool Equipment, Barn) polling every 60s
- Maintains local SQLite database of zone state, tank level, watering events, and per-zone learning coefficients
- Integrates Open-Meteo weather/ET data with Penman-Monteith calculations
- Implements SMS observation feedback loop (Twilio) for zone health ratings
- Takes scheduling control via Hydrawise setzone API (Phase 4+)
- Web dashboard for monitoring and control (Phase 4+: Next.js + Supabase + Vercel)

**Property Details:**
- Location: Loomis, CA (38.8024, -121.1964, 122m elevation)
- 22 irrigation zones across 3 controllers
- 1,725 gal tank (981 gal usable), ditch water season Apr 15 - Oct 15
- Always-on Lenovo Legion host, pm2 process manager

2. **Hydrawise controllers serialize valves** → one zone per controller at a time, Garage + Pool Equipment share serialized timeline (attribution group)
3. **Project deliberately outside OneDrive** → SQLite WAL files conflict with OneDrive sync
4. **Phase 6 ditch health check needs redesign** → capping Garage Z5 closed off planned mechanism
